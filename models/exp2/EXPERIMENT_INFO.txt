EXPERIMENT 2: Improved Models with Tuning + XGBoost + Lag Features
================================================================================

OBJECTIVE
---------
Improve upon baseline (exp1) performance through:
  1. Hyperparameter optimization
  2. Advanced algorithms (XGBoost)
  3. Temporal feature engineering (lag features)

IMPROVEMENTS OVER EXPERIMENT 1
-------------------------------
1. **Hyperparameter Tuning**
   - GridSearchCV with 3-fold cross-validation
   - Tested combinations of n_estimators, max_depth, learning_rate, etc.
   
2. **XGBoost Algorithm**
   - Gradient boosting typically outperforms Random Forest
   - Better handling of feature interactions
   - Grid search across: n_estimators=[200,300], max_depth=[10,15,20],
     learning_rate=[0.05,0.1], subsample=[0.8,1.0], colsample=[0.8,1.0]

3. **Tuned Random Forest**
   - Increased trees: 300 (vs 200 in exp1)
   - Deeper trees: max_depth=30 (vs 20 in exp1)
   - More flexible splitting parameters

4. **Lag Features** (NEW)
   - PM_lag1: Yesterday's PM values
   - PM_rolling3: 3-day rolling average
   - PM_rolling7: 7-day rolling average
   - AOD_rolling3: 3-day AOD average
   - NO2_rolling3: 3-day NO2 average
   
   Total features: ~31 (vs 23 in exp1)

APPROACH
--------
- Train BOTH XGBoost and tuned Random Forest for PM2.5 and PM10
- Compare validation performance
- Select best model for each pollutant
- Models may differ (e.g., XGBoost for PM2.5, RF for PM10)

EXPECTED IMPROVEMENTS
---------------------
Target R² gains over exp1:
  - PM2.5: 0.45 → 0.55-0.65 (+10-20%)
  - PM10: 0.55 → 0.60-0.70 (+5-15%)

NOTES
-----
- Lag features require temporal ordering by station
- First day for each station has NaN lags → filled with median
- GridSearch may take 15-30 minutes (48 combinations × 3 folds)
